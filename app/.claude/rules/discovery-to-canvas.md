# Discovery Content Goes to Docs

During discovery conversations, **write substantive content to discovery docs first, not chat.**

- When brainstorming produces structured output (clusters, questions, frameworks, options), write it to the relevant doc in `docs/discovery/`
- Open the doc in the canvas so the human follows along there
- Chat is for steering, reacting, and short responses — not for long-form discovery content
- If you're about to write more than ~10 lines of structured thinking in chat, it belongs in a doc instead

This keeps the discovery artifacts as the source of truth, not buried in conversation history.
